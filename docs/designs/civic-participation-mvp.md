## Vision

### 10x Check
The 10x version is a civic operating system where every piece of UK government data has a comment thread, every budget line item has community proposals, every MP vote generates a diff that citizens discuss in real-time, and the platform becomes the default place UK citizens go to understand and engage with how their country runs. GitHub Issues for the entire UK government, not just council budgets.

## Review Findings

### Architecture
- Clean separation: static V2.0 data + new Supabase participation layer
- Rollback: 5/5 reversibility (delete Supabase project, remove routes)
- No coupling concerns
- Full data model in design doc (users, proposals, votes, comments, civic_diffs tables)

### Security
- XSS: plain text only for user-generated input (proposals, comments). System-rendered output (OG cards, widgets) uses controlled templates.
- Auth: Supabase JWT + RLS policies enforce per-user access
- Vote manipulation: email verification required. CAPTCHA deferred.
- Public API: read-only, no auth required, rate-limited

### Edge Cases Resolved
- Comment threading: max 3 levels (Reddit mobile pattern)
- Proposals pagination: cursor-based on `created_at` + `id` composite, 20 per page
- Moderation: async Haiku classification (spam/hate/ok labels, auto-hide on score > 0.7, community flag threshold of 3 flags = hidden pending review). Fallback: if Haiku unavailable, post stays visible, community flagging only.
- Anonymous drafts: localStorage with 24hr TTL, no server storage, no spam vector
- Email digest: batched 100 users per Edge Function invocation, pg_cron trigger, max 50 follows per user
- OG images: @vercel/og with Satori, truncate titles at 120 chars in image

### Error Map (top 5 critical paths, full map in design doc)
1. Haiku API unavailable → post appears, community flagging only
2. Supabase connection failure → 500 with retry message, existing V2.0 data unaffected
3. Double-vote attempt → upsert changes vote direction (unique constraint)
4. Magic link email delivery failure → log, retry once, user sees "check your email"
5. OG image generation timeout → return default CivAccount fallback image
